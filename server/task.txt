URL SHORTNER -

takes in a valid UURL and returns a shorthand URL, redirecting the user to the previously provided URL.

Also, keep track of tatal visits/clicks on the URL.

ROUTES - 
POST /URL - Generates a new short URL and returns the shorthand URL in the format example.com/random-id.

GET /:id - Redirects the user to the original URL.

GET /URL/analytics/:id - Returns the clicks for the provided short id.


NOTE : Specific routes -> first  
       Dynamic routes -> last
       `locals` -> object that holds the data your server sends to the template.
       mongoose.Schema.Types.ObjectId -> reference to the user who created this user (for admin purposes)

4.  POST → Redirect → GET (PRG Pattern)

    The PRG (Post-Redirect-Get) pattern prevents duplicate form submissions by redirecting the user after a POST request instead of directly rendering a page.

    Never render directly after a successful POST request. If refreshing, can cause duplicate actions.
